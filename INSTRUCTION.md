# Пошаговая инструкция по запуску системы управления BLDC-двигателем

## Требования
- Python 3.8 или выше
- Установленные библиотеки: `python-can`, `matplotlib`

## Установка зависимостей (если еще не установлены)
```powershell
pip install python-can matplotlib
```

---

## Шаг 1: Откройте PowerShell
Нажмите `Win + X` и выберите **Windows PowerShell** или **Terminal**

## Шаг 2: Перейдите в директорию с файлом
Предполагается, что файл `bldc_motor_control.py` находится на рабочем столе.
Введите команду:

```powershell
cd C:\Users\User\Desktop
```

> **Важно:** Замените путь на актуальный, если файл находится в другом месте.

## Шаг 3: Проверьте наличие файла
Убедитесь, что файл существует:

```powershell
dir bldc_motor_control.py
```

Вы должны увидеть файл в списке.

## Шаг 4: Запустите программу в режиме симуляции

Для тестирования без физического оборудования используйте флаг `--simulate` или `-s`:

```powershell
python bldc_motor_control.py --simulate
```

Или короткая версия:

```powershell
python bldc_motor_control.py -s
```

## Шаг 5: Наблюдайте за графиками

После запуска должно открыться окно с двумя графиками:

1. **Верхний график (D-Q Frame Currents)**: 
   - Синяя линия: ток Id
   - Красная линия: ток Iq

2. **Нижний график (Three-Phase Currents)**:
   - Зеленая линия: ток Ia
   - Фиолетовая линия: ток Ib
   - Голубая линия: ток Ic

В левом верхнем углу отображаются текущие значения токов в реальном времени.

## Шаг 6: Остановка программы
Закройте окно графика мышкой или нажмите `Ctrl + C` в окне PowerShell.

---

## Дополнительные параметры запуска

### Запуск с указанием CAN-интерфейса (для реального оборудования)
Если у вас подключен CAN-адаптер:

```powershell
python bldc_motor_control.py --channel can0 --bitrate 500000
```

### Просмотр всех доступных опций
```powershell
python bldc_motor_control.py --help
```

## Решение проблем

### Проблема: "module 'can' has no attribute 'interface'"
**Решение:** Убедитесь, что установлена последняя версия python-can:
```powershell
pip install --upgrade python-can
```

### Проблема: График не отображается или пустой
**Решение:**
1. Убедитесь, что используете флаг `--simulate` для тестирования без оборудования
2. Проверьте, что matplotlib установлен корректно:
   ```powershell
   pip install --upgrade matplotlib
   ```
3. Попробуйте увеличить интервал обновления:
   ```powershell
   python bldc_motor_control.py --simulate
   ```
   (интервал по умолчанию 50 мс уже оптимален)

### Проблема: Окно графика сразу закрывается
**Решение:** Запустите PowerShell от имени администратора или проверьте настройки безопасности Windows.

### Проблема: Ошибка импорта tkinter
**Решение:** Установите tkinter (обычно входит в состав Python для Windows):
```powershell
pip install tk
```
Или переустановите Python с опцией tcl/tk.

## Подключение реального двигателя KMTech MG4010E v3

1. Подключите CAN-адаптер к компьютеру
2. Установите драйверы адаптера (IXXAT, Vector, Peak и т.д.)
3. Узнайте имя CAN-интерфейса в системе
4. Запустите программу без флага `--simulate`:
   ```powershell
   python bldc_motor_control.py --channel <имя_интерфейса> --bitrate 500000
   ```

> **Примечание:** CAN ID (0x201 для получения данных о токах) могут отличаться в зависимости от конфигурации двигателя. Сверьтесь с документацией KMTech MG4010E v3.

---

## Формат данных CAN

Программа ожидает следующие данные в CAN-сообщениях (ID 0x201):
- Байты 0-1: Ток Id (signed int16, масштаб 0.01A)
- Байты 2-3: Ток Iq (signed int16, масштаб 0.01A)
- Байты 4-5: Ток Ia (опционально)
- Байты 6-7: Ток Ib (опционально)

Если данные о фазных токах не передаются, они рассчитываются программно.
